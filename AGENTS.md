## Tone
Пиши простым русским языком, коротко и по делу.

## Landing Rules
- Каждый блок отвечает на вопрос «что получит человек?»
- Без воды и штампов
- Один главный CTA на экран

## Anti-Slop
Запрещены слова «инновационный», «революционный», «уникальный» без фактов.
Если фраза звучит как шаблон нейросети — перепиши.

## Done Criteria
- Мобильная версия читается без зума
- Оффер понятен за 3 секунды
- Есть конкретный следующий шаг (CTA)

## Trigger Rule
Если меняются оффер/CTA/структура страницы — обязательно запускай recurring-задачу Landing Polish из RECURRING_TASKS.md.

## Git & Deploy
- После успешной итерации по лендингу: `git add` → осмысленный коммит → `git push`.
- Проект привязан к Vercel: после пуша должна уходить новая прод-версия.
- Не коммить `.vercel`, `.env*` и любые секреты.

## Hero & CTA
- Hero обязан содержать: короткий оффер (<10 слов), расшифровку в 1–2 предложениях, один главный CTA и факт доверия (цифра/факт).
- На мобиле главный CTA на всю ширину и легко жмётся большим пальцем.
- В одном экране — только один приоритетный CTA, остальные второстепенные делай визуально слабее.

## Content Specificity
- В каждом смысловом блоке хотя бы один конкретный факт или число.
- Если фраза не отвечает на «и что?», перепиши в формате «[действие] → [результат]».
- Без расплывчатых обещаний вроде «повышает эффективность» без пояснения «насколько и за счёт чего».

## Social Proof
- Обещания усиливай: логотипами, цитатами гостей/партнёров или кейсами, но без выдуманных «топ‑брендов».
- Хороший соцдоказ: лицо, имя, роль, компания и конкретный результат.
- Если нет реальных данных — лучше честно обойтись без соцдоказательств.

## Mobile & Density
- Проверяй на ширине типа iPhone SE: заголовок в 2–3 строки максимум, текст в 3–4 строки без зума.
- Базовый размер текста не меньше 14px, кликабельные элементы не ниже 44px по высоте.
- Избегай длинных простыней текста — дроби на короткие абзацы с понятными заголовками.

## Structure Changes
- При изменении оффера/CTA/структуры фиксируй в описании коммита, какую гипотезу по конверсии проверяешь.
- Перед такими изменениями перечитай этот файл и `RECURRING_TASKS.md`.
